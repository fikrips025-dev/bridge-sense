Metodologi & Parameter
📐 SCS → ECS Projection

Akselerometer smartphone merekam getaran dalam Smartphone Coordinate System (SCS) yang bergantung orientasi fisik perangkat. Bridge-Sense memproyeksikannya ke Earth Coordinate System (ECS) via matriks rotasi dari DeviceOrientationEvent (α, β, γ). Hanya komponen vertikal Z yang diteruskan ke buffer.

a_ECS = R(α,β,γ) · a_SCS
a_linear = a_total − g_vector
g_vector ≈ [0, 0, 9.80665] m/s²

🧹 Garbage Collection

Setelah setiap transmisi, array buffer dikosongkan eksplisit (buffer.length = 0) untuk mencegah akumulasi heap memory yang dapat menyebabkan crash pada browser perangkat kelas menengah ke bawah.

📡 Validasi GPS Haversine

Koordinat pengguna divalidasi terhadap geofence jembatan menggunakan Formula Haversine. Transmisi hanya diterima jika pengguna berada dalam radius valid di atas struktur jembatan.

ℹ️ Tentang Proyek

Bridge-Sense adalah prototipe riset Mobile Crowdsensing untuk Structural Health Monitoring (SHM) berbiaya rendah. Memanfaatkan Hukum Bilangan Besar: agregasi data masif dari banyak pengguna menekan noise acak, menyisakan frekuensi natural absolut struktur jembatan.

📳
Izin Sensor Gerak

Bridge-Sense memerlukan akses ke akselerometer dan giroskop perangkat untuk mengukur getaran jembatan secara real-time.

⚠️ Pada iOS 13+, izin sensor gerak harus diminta secara eksplisit. Data sensor tidak pernah dikirim tanpa konfirmasi Anda.
